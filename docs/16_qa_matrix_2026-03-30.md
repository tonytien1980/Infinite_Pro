This file records the latest manual QA pass for the current Infinite Pro workbench.

Verification posture:
- real backend + real frontend
- browser-driven interaction on local runtime
- non-destructive actions prioritized first
- destructive / cleanup actions intentionally excluded unless explicitly requested

Environment used:
- frontend: `http://127.0.0.1:3001`
- backend: `http://127.0.0.1:8010/api/v1`
- database: local `acceptance.db`

Important runtime note:
- When building frontend for local QA, keep `NEXT_PUBLIC_API_BASE_URL=http://127.0.0.1:8010/api/v1`
- If omitted, the client bundle can fall back to `:8000`, which may point at a different backend/runtime
